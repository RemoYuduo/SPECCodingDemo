# 企业内部福利商城 - 后端架构信息

## 目录结构
```
backend/
├── src/
│   ├── controllers/        # 控制器层
│   ├── services/           # 业务逻辑层
│   ├── routes/             # 路由层
│   │   ├── auth.js         # 认证路由(已实现)
│   │   ├── user.js         # 用户路由(已实现)
│   │   ├── admin.js        # 管理员路由(已实现)
│   │   └── health.js       # 健康检查路由(已实现)
│   ├── middleware/         # 中间件
│   │   ├── auth.js         # 认证中间件(已实现)
│   │   └── error.js        # 错误处理中间件(已实现)
│   ├── config/             # 配置文件
│   │   ├── database.js     # 数据库配置(已实现)
│   │   └── app.js          # 应用配置(已实现)
│   ├── database/           # 数据库相关
│   │   └── init.js         # 数据库初始化(已实现)
│   └── utils/              # 工具函数
├── .env                    # 环境变量配置
├── package.json            # 依赖配置(已实现)
└── server.js               # 服务器入口文件(已实现)
```

## 模块外部特性

### 认证模块 (auth/) - 已实现
- 登录接口：验证用户身份，返回JWT令牌
- 验证中间件：验证请求中的JWT令牌
- 权限控制：区分普通用户和管理员权限

### 用户模块 (user/) - 已实现基础功能
- 用户信息接口：获取和更新用户个人资料
- 商品浏览接口：获取商品列表和商品详情
- 测试商品接口：返回模拟商品数据用于连通性测试

### 管理员模块 (admin/) - 已实现基础功能
- 管理员测试接口：验证管理员身份
- 系统状态接口：获取系统基本信息

### 健康检查模块 (health/) - 已实现
- 服务健康检查：检查服务器运行状态
- 数据库状态检查：检查数据库连接和表状态

### 中间件 (middleware/) - 已实现
- 认证中间件：验证JWT令牌和用户身份
- 错误处理中间件：统一处理请求错误

### 配置模块 (config/) - 已实现
- 数据库配置：SQLite数据库连接和初始化
- 应用配置：端口、环境变量等应用设置

### 数据库模块 (database/) - 已实现
- 数据库初始化：创建基础表结构和插入测试数据
- 数据模型：用户、商品、订单、购物车和积分记录表

### 服务器入口 - 已实现
- Express服务器配置
- 路由注册和中间件应用
- 数据库初始化和错误处理
