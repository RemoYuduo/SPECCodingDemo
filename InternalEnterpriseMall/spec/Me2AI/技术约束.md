# 企业内部福利商城 Me2AI



# 技术架构

## 1. 前端架构
- 使用Vue 3框架
- 组件化开发，提高代码复用性
- 响应式设计，适配PC和移动端
- 使用Pinia进行状态管理

### 2. 后端架构
- 采用微服务架构
- 核心服务拆分：用户服务、商品服务、订单服务、支付服务、积分服务
- Spring Cloud Gateway统一管理请求路由
- 服务间通信采用RESTful API

### 3. 数据存储
- SQLite

### 4. 集成方案
- 企业SSO系统集成（OAuth2）
- 企业HR系统数据同步（员工信息、部门信息）
- 企业支付系统集成

## 核心数据库设计

### 1. 用户相关
- 用户表（与企业用户表关联）
- 部门表
- 积分账户表
- 积分变动记录表

### 2. 商品相关
- 商品分类表
- 商品信息表
- 商品库存表

### 3. 订单相关
- 订单表
- 订单商品表
- 购物车表

### 4. 支付相关
- 支付记录表
- 积分使用记录表

## 项目里程碑

### 第一阶段：基础平台（可独立运行测试）
**目标**：搭建基础框架，实现用户管理和商品浏览功能
- **前端**：
  - 管理后台基础框架搭建（登录、权限控制、基础布局）
  - 员工商城H5基础框架搭建（首页、商品列表、商品详情）
- **后端**：
  - 用户服务（员工登录、个人信息、部门管理）
  - 商品服务（商品分类、商品列表、商品详情）
  - 基础认证授权（JWT令牌、权限控制）
- **数据库**：
  - 用户表、部门表
  - 商品分类表、商品表
- **测试点**：
  - 员工可以登录系统
  - 可以浏览商品列表和详情
  - 管理员可以管理用户和商品

### 第二阶段：交易系统（可独立运行测试）
**目标**：实现完整的购物、支付和订单管理功能
- **前端**：
  - 购物车功能（添加商品、数量修改、删除）
  - 订单创建功能（商品选择、地址填写、支付方式选择）
  - 个人中心（订单查询、个人信息）
- **后端**：
  - 订单服务（订单创建、订单查询、订单状态管理）
  - 支付服务（支付接口、支付回调、支付状态查询）
  - 积分服务（积分查询、积分扣减、积分记录）
- **数据库**：
  - 订单表、订单商品表、购物车表
  - 支付记录表、积分账户表、积分变动记录表
- **测试点**：
  - 员工可以将商品加入购物车
  - 员工可以创建订单并完成支付
  - 系统可以正确处理积分和现金支付
  - 订单状态可以正确流转

### 第三阶段：物流与管理（可独立运行测试）
**目标**：实现物流配送和系统管理功能
- **前端**：
  - 物流查询功能（订单物流跟踪、配送地址管理）
  - 管理后台高级功能（订单管理、物流管理、数据统计）
- **后端**：
  - 物流服务（配送地址管理、物流信息跟踪）
  - 系统集成（企业HR系统、企业支付系统、企业SSO）
  - 基础监控和日志
- **数据库**：
  - 配送地址表、物流信息表
- **测试点**：
  - 员工可以管理配送地址
  - 员工可以查询订单物流状态
  - 管理员可以管理订单和物流
  - 系统可以与企业系统正常交互

### 第四阶段：优化与完善（可独立运行测试）
**目标**：系统性能优化、安全加固和用户体验提升
- **前端**：
  - 界面优化和响应式设计改进
  - 性能优化（资源压缩、懒加载）
- **后端**：
  - 性能优化（缓存策略、数据库优化、接口优化）
  - 安全加固（权限细化、敏感信息加密、防护增强）
  - 系统监控（性能监控、异常监控、日志分析）
- **测试点**：
  - 系统响应速度明显提升
  - 系统安全性经过测试验证
  - 系统可以稳定运行
  - 异常情况可以及时告警和处理
